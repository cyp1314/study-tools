## 一、数据库表设计（新增 4 张表）

### 1. `users`（已有，需扩展字段）
```sql
ALTER TABLE users ADD COLUMN phone VARCHAR(20) DEFAULT '';

-- 新增微信字段
ALTER TABLE users ADD COLUMN unionid VARCHAR(64) DEFAULT '';
```

### 2. `user_points` 用户积分表
```sql
CREATE TABLE user_points (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL UNIQUE,
  balance INT NOT NULL DEFAULT 0 COMMENT '当前积分余额',
  total_earned INT NOT NULL DEFAULT 0 COMMENT '累计获得积分',
  total_spent INT NOT NULL DEFAULT 0 COMMENT '累计消耗积分',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_user (user_id)
);
```

### 3. `point_logs` 积分变更记录表
```sql
CREATE TABLE point_logs (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  type ENUM('sign_in','recharge','generate','admin_adjust','other') NOT NULL COMMENT '变更类型',
  amount INT NOT NULL COMMENT '变更数量(正=获得,负=消耗)',
  balance_after INT NOT NULL COMMENT '变更后余额',
  remark VARCHAR(255) DEFAULT '' COMMENT '备注(如签到第3天/生成涂色绘本)',
  ref_id VARCHAR(100) DEFAULT '' COMMENT '关联业务ID(如image_records.id)',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_user (user_id),
  INDEX idx_type (type),
  INDEX idx_created (created_at)
);
```

### 4. `sign_in_records` 签到记录表
```sql
CREATE TABLE sign_in_records (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  sign_date DATE NOT NULL COMMENT '签到日期',
  continuous_days INT NOT NULL DEFAULT 1 COMMENT '当前连续签到天数',
  earned_points INT NOT NULL DEFAULT 0 COMMENT '本次获得积分',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_user_date (user_id, sign_date),
  INDEX idx_user (user_id),
  INDEX idx_date (sign_date)
);
```

### 5. `sign_in_config` 签到奖励配置表
```sql
CREATE TABLE sign_in_config (
  id INT PRIMARY KEY AUTO_INCREMENT,
  continuous_days INT NOT NULL COMMENT '连续第N天',
  reward_points INT NOT NULL COMMENT '奖励积分',
  is_active TINYINT DEFAULT 1 COMMENT '是否启用',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_days (continuous_days)
);
-- 默认配置: 第1天5分, 第2天10分, 第3天15分...第7天50分, 之后每天50分
```

### 6. `recharge_orders` 充值订单表
```sql
CREATE TABLE recharge_orders (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  order_no VARCHAR(64) NOT NULL UNIQUE COMMENT '订单号',
  points INT NOT NULL COMMENT '充值积分数',
  amount DECIMAL(10,2) NOT NULL COMMENT '支付金额(元)',
  status ENUM('pending','paid','failed','refunded') DEFAULT 'pending',
  payment_method VARCHAR(30) DEFAULT '' COMMENT '支付方式(wechat等)',
  paid_at DATETIME DEFAULT NULL COMMENT '支付时间',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_user (user_id),
  INDEX idx_order (order_no),
  INDEX idx_status (status)
);
```

## 三、API 设计

### 登录模块 `/api/v1/auth`
- `POST /login` — 微信登录(body: {code}) → 返回 token + 用户信息
- `POST /login/phone` — 绑定手机号(body: {phone, code})

### 用户模块 `/api/v1/user` (需auth)
- `GET /profile` — 获取用户信息+积分余额
- `PUT /profile` — 更新昵称/头像

### 积分模块 `/api/v1/point` (需auth)
- `GET /balance` — 查询积分余额
- `GET /logs` — 积分变更记录(分页)
- `GET /recharge/packages` — 充值套餐列表

### 签到模块 `/api/v1/signin` (需auth)
- `POST /` — 签到(计算连续天数→查配置→加积分→写日志)
- `GET /status` — 今日签到状态+连续天数+签到日历
- `GET /config` — 签到奖励配置

### 充值模块 `/api/v1/recharge` (需auth)
- `POST /create` — 创建充值订单
- `POST /notify` — 支付回调(微信支付通知)

## 四、核心业务逻辑

### 微信登录流程
1. 小程序调用 `wx.login()` 获取 code
2. 后端用 code 调用微信 `code2Session` 接口获取 openid + session_key
3. 查找/创建用户 → 签发 JWT → 返回

### 积分扣减（生成图片时）
1. 查询用户积分余额
2. 余额不足 → 抛出错误
3. 扣减积分 → 写 point_logs → 更新 user_points
4. 在 image controller 的 generate 方法中调用 `pointService.deduct()`

### 签到流程
1. 检查今日是否已签到
2. 查昨日签到记录，计算连续天数
3. 查 sign_in_config 获取对应奖励积分
4. 写 sign_in_records + 加积分 + 写 point_logs
