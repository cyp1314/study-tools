# 童学绘 - 儿童学习App 需求文档与开发计划

---

## 一、项目概述

**项目名称**：童学绘（Study Tools Kids）

**项目定位**：面向 3-12 岁儿童的趣味学习工具，以"涂色 + 书写 + 练习 + 打印"为核心，让孩子在玩中学，家长一键生成可打印的学习材料。

**目标平台**：微信小程序（主要），打印/下载功能保存为图片到相册

**技术栈**：

- 客户端：UniApp (Vue 3 + TypeScript) + Wot UI + UnoCSS + Pinia
- 服务端：Koa + MySQL + AI API
- 存储：MySQL（结构化数据）+ OSS（图片/素材资源）

### 客户端核心组件

1. **Canvas 涂色引擎** - 支持区域填充、自由画笔、颜色选择
2. **格子渲染器** - 田字格/四线三格/作文格/竖式格等统一渲染
3. **图形生成器** - 时钟/数轴/几何图形等动态生成
4. **图片导出器** - Canvas → 图片 → 保存到相册

### 服务端核心模块

1. **素材管理** - 线稿SVG、汉字笔画数据、诗词词库的CRUD
2. **题目生成** - 算术题/英语题的模板化生成逻辑
3. **AI 服务** - 大模型API封装（智能出题、错题生成、讲解生成）
4. **图片渲染** - 服务端Canvas/Puppeteer生成A4尺寸高清图片（备选方案）

### 数据库核心表

- `users` - 用户表（家长/孩子）
- `categories` - 功能分类
- `materials` - 素材库（线稿/笔画/词库等）
- `works` - 用户作品记录
- `exercises` - 练习题库
- `error_books` - 错题本
- `reward_records` - 奖励积分记录

## 六、技术要点与风险

### 核心技术点

1. **Canvas 涂色引擎**：微信小程序 Canvas 2D API，区域填充需实现洪水填充算法（Flood Fill）
2. **格子渲染**：统一封装 GridRenderer，支持田字格/四线三格/米字格/作文格等
3. **图片导出**：Canvas → `canvasToTempFilePath` → `saveImageToPhotosAlbum`
4. **AI 集成**：后端统一代理大模型API，Prompt模板化管理，结构化JSON输出
5. **汉字笔顺**：使用 hanzi-writer 数据或自建笔画数据库

### 风险与应对

| 风险                 | 应对方案                           |
| -------------------- | ---------------------------------- |
| 小程序Canvas性能限制 | 控制画布尺寸，分片渲染，离屏Canvas |
| 涂色区域填充卡顿     | 优化洪水填充算法，限制画布分辨率   |
| AI生成内容质量不稳定 | 模板约束输出格式，多次重试机制     |
| 素材库版权问题       | 使用开源/CC协议素材，或AI生成线稿  |
| 小程序包体积限制     | 使用分包加载，素材走OSS远程加载    |
