## 1. 设计规范

### 1.1 色彩体系

```
主色：    #FF6B9D（温暖粉红）- 主要按钮、强调色
辅色：    #4ECDC4（清新蓝绿）- 次要按钮、图标
强调色：  #FFE66D（明亮黄）  - 徽章、高亮、奖励
背景色：  #FFF5F7（淡粉白）  - 页面背景
卡片色：  #FFFFFF（纯白）    - 内容卡片
主文本：  #2C3E50（深蓝灰）  - 标题、正文
次文本：  #7F8C8D（中灰）    - 说明文字
```

### 1.2 字体规范

```
标题：    32px / Bold / 2C3E50
副标题：  24px / Semibold / 2C3E50
正文：    16px / Regular / 2C3E50
说明：    14px / Regular / 7F8C8D
标签：    12px / Medium / 7F8C8D
```

### 1.3 圆角规范

```
大圆角：  16px - 卡片、区块
中圆角：  12px - 按钮、输入框
小圆角：  8px  - 标签、徽章
圆形：    50%  - 头像、颜色选择器
```

### 1.4 间距规范

```
页面边距：  16px
卡片间距：  16px
元素间距：  12px / 8px / 4px
网格间距：  12px
```

## 8. 交互说明

### 8.1 通用交互

1. **页面切换**
   - 右滑返回上一页
   - 底部 TabBar 切换无动画
   - 页面内 Tab 切换使用滑动动画

2. **加载状态**
   - 首次加载：全屏 Loading
   - 下拉刷新：顶部刷新指示器
   - 上拉加载更多：底部 Loading + 提示文字

3. **空状态**
   - 无数据：显示插画 + 引导文字 + 操作按钮
   - 无网络：显示网络错误 + 重试按钮
   - 加载失败：显示错误信息 + 重试按钮

4. **反馈提示**
   - 成功：Toast 提示（1.5s）
   - 失败：Toast 提示（2s）
   - 警告：Modal 确认
   - 进度：Progress 条

### 8.2 涂色交互

```
触摸开始 → 记录起点坐标
    ↓
触摸移动 → 绘制路径（节流 16ms）
    ↓
触摸结束 → 保存路径到历史
    ↓
撤销/重做 → 从历史栈恢复
```

**性能优化**：
- 使用 Canvas 2D API（非 WebGL）
- 路径绘制节流到 60fps
- 限制画布最大分辨率
- 离屏 Canvas 预渲染

### 8.3 手势操作

| 手势 | 操作 | 说明 |
|------|------|------|
| 单击 | 选择颜色/工具 | 基础操作 |
| 长按 | 显示快捷菜单 | 工具选项 |
| 双指捏合 | 缩放画布 | 0.5x - 3x |
| 双指拖动 | 平移画布 | 边界限制 |
| 右滑 | 撤销 | 快捷操作 |
| 左滑 | 重做 | 快捷操作 |

### 9.3 核心 API 调用

```typescript
// 生成图片
async function generateImage(type: string, params: any) {
  const res = await uni.request({
    url: `${BASE_URL}/api/image/generate`,
    method: 'POST',
    data: { type, params },
  });
  return res.data;
}

// 保存图片到相册
async function saveToAlbum(imageUrl: string) {
  const { tempFilePath } = await uni.canvasToTempFilePath({
    canvasId: 'drawing',
  });
  
  await uni.saveImageToPhotosAlbum({
    filePath: tempFilePath,
  });
}

// 上传作品
async function uploadWork(workData: any) {
  const res = await uni.uploadFile({
    url: `${BASE_URL}/api/work/upload`,
    filePath: workData.imagePath,
    name: 'image',
    formData: workData.metadata,
  });
  return res.data;
}
```

### 9.4 性能优化

1. **Canvas 优化**
   - 使用离屏 Canvas 预渲染复杂图形
   - 限制画布最大宽度为 750px
   - 使用 `will-change` 提示浏览器优化

2. **图片优化**
   - 素材走 OSS CDN
   - 使用 WebP 格式（支持时）
   - 实现懒加载和预加载

3. **分包优化**
   - 按需加载功能模块
   - 预下载常用分包
   - 使用 `preloadRule` 配置

4. **内存管理**
   - 及时释放不用的 Canvas 上下文
   - 限制历史记录栈大小（最多50步）
   - 使用对象池复用频繁创建的对象
